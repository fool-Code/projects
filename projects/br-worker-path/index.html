Brazilian Workers Commuting

Undergraduate research on Brazilian commuter networks

Python NetworkX Numpy Geopandas

Project Overview

As part of my undergrad research, I modelled a city's commuter system as a graph: stations as nodes, routes as edges. Using Python + NetworkX, I measured how the average shortest path changes with network size (log of nodes vs. path length) and ran simple regressions to spot patterns.

It was a quick dive into transport analysis that taught me data cleaning, graph metrics, and how to turn raw travel data into real insights.

Technical Implementation

This project analyses urban commuter networks as graphs for an undergraduate research study. Stations or stops are treated as nodes and routes as edges, allowing the measurement of network connectivity and travel efficiency.

Architecture Overview

Edge List (region data) → Build Graph (NetworkX) → Compute Metrics (log(Nodes), Average Shortest Path) → Linear Regression → Save & Visualise Results

Python Analysis Code

import networkx as nx
import numpy as np
from scipy.stats import linregress
import pickle

def build_graph(edge_list):
    G = nx.Graph()
    G.add_edges_from(edge_list)
    return G

def graph_stats(G):
    avg_path = nx.average_shortest_path_length(G)
    log_nodes = np.log(G.number_of_nodes())
    return log_nodes, avg_path

# Example workflow
all_regions = [edges_region1, edges_region2, ...]
logn_vs_avg = np.asarray([
    graph_stats(build_graph(edges))
    for edges in all_regions
])

# Fit a linear regression
slope, intercept, r, p, stderr = linregress(logn_vs_avg[:,0], logn_vs_avg[:,1])

with open("data/logn_vs_avgpath.pkl", "wb") as f:
