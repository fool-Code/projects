Denoising with PyTorch

Exploring image denoising with PyTorch and Gated Attention Units (GAUs)

PyTorch Python imageio Neural Networks CIFAR-10

Overview

This project explores image denoising using a small PyTorch model that incorporates Gated Attention Units (GAUs). The objective was primarily exploratory: generate noisy images using Gaussian and Poisson noise, train a model to recover the original clean versions, and evaluate performance visually.

The work began from code suggested by ChatGPT; through debugging and gradually understanding each component, the project became an opportunity to explore PyTorch more systematically—custom datasets, training loops, and attention mechanisms. This experience motivated further study through Deep Learning with PyTorch and a Mathematical Statistics course.

Key Features

Exploratory Denoising

Implemented a curiosity-driven denoising pipeline using GAUs.

Noise Simulation

Added Gaussian and Poisson noise to CIFAR-10 and Kodak dataset images.

Custom Architecture

Built a lightweight encoder–decoder network enhanced with GAU blocks.

Visual Evaluation

Compared noisy, clean, and reconstructed images for qualitative assessment.

Learning-Focused

Aimed at understanding PyTorch internals, not maximizing performance.

Hardware Constraints

Experiments performed on older hardware (i5-7200U, GTX 940MX, 16GB RAM).

Technical Implementation

The denoising network receives noisy CIFAR-10 images and attempts to reconstruct cleaner versions. The architecture follows a simple pattern: convolutional encoder, GAU blocks, and a decoder. The model was trained on a small dataset split due to hardware limitations.

Architecture Overview

Input (Noisy Image) → Encoder → GAU Blocks → Decoder → Output (Denoised Image)

Sample Input/Output

Noisy: tensor([[[0.12, 0.34, 0.27], ...]]) Denoised: tensor([[[0.10, 0.32, 0.25], ...]]) Clean: tensor([[[0.11, 0.33, 0.26], ...]])

Results

As expected, the model’s performance was limited. Results were not sharp enough to be considered clean reconstructions — largely due to training on a small sample set, necessitated by hardware limitations.
